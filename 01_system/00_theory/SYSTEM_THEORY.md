# Relation System Theory

1. 중심 관계축은 `AI relation → 승이 relation → GitHub`다.
2. `Relation = System + DB + System–DB Contract`다.
3. `Local.Zero`는 절대기원이 아니라 양방향 분석을 위해 선택된 분석기점이다.
4. `Result`는 이전 과정의 Fruit이자 다음 `Question`의 Seed다.
5. Reverse Reading은 Reverse Causality가 아니다.
6. Relation is not merge; relation is interconnecting.
7. Source precedes interpretation.
8. Comparison precedes local closure.
